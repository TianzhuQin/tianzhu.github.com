Abstract¶

The essay hererby provides Reinforcement Learning strategies to make trading in stock market. Deep Direct Reinforcement Network, which utilizes Deep Learning to find the tendency of stock prices and Recurrent Neural Network to excute buy/sell action, performs robustly that constantly makes profits in various combinations of hyper-parameters. The essay implement differing kinds of Deep Learning, say Deep Fully Connected Network, Convolution Neural Network and Recurrent Neural Network to predict stock prices in the future. All of them, with sensitive analysis, have good performance. The essay also implements Deep Q Learning, which decides the action by monitoring the Q-value of the buy/sell option it will take. The result is not as good, mainly because that Q-table is not easily gotten for time series data, although evident upward tendency is shown when training. Overall, the work makes comparative analysis of the two models, DDRN and DQN, and gives suggestions on trading with Reinforcement Learning.

1. Introduction¶

Stock market is famous for its unpredictable characteristic, and the kinds of trading strategies are so massive that one can hardly learn all of them in a short period (Heaton et al., 2017).

The two pain points of trading in this market are 1) how to predict further variation of the stock; 2) how to perform in terms of the corresponding variation. The stock price prediction is very hard due to the uncertainty of the stock itself, with many outer issues affecting the market (Heaton et al., 2017; Deng et al., 2016; Kimoto et al., 1990; Schneburg & Eberhard, 1990). As for actions, methods by tradional financial statistics roughly calculate the variation of the portfolio and distribute the investment well to minimize that, which does not certainy results to a high return at last (Jondeau et al., 2006).

Fortunately, Deep Learning methods such as Convolution Neural Network, Recurrent Neural Network or even just Deep Fully Connected Network, are very useful for prediction, especially the time-series data, which is correlated among stocks. What is more fortunate, Reinforcement (which can be showed as Recurrent) thoughts are helpful to represent the long-time trading. On the other words, action to take each trading day can be acquired by these minds (Fischer & Krauss, 2018; Heaton et al., 2017; Deng et al., 2016; Choi et al., 2009).

To take the research, the essay hereby utilize two main different trading algorithm, Deep Direct Reinforcement Network and Deep Q Network (Mnih et al., 2015; Heaton et al., 2017), while each main algorithm has differing forms to dive further, so as to acquire outcomes, as excellent as possible and as stable as possible. All evaluations and sensitivity-analysis in need follow the corresponding algorithms.

def build_batch(data, step, lag, num_batches):
    indices = np.arange(lag , len(data) - step + 1) # should be steps ahead of the last one
    np.random.shuffle(indices) # shuffle the id rather than data in order to save memory
    batches = np.zeros([num_batches, lag + step, num_stocks])
    
    for i in range(num_batches):
        index = indices[i]
        batch = data[(index - lag):(index + step)] # if index=2, lag=2, step=6: data[0, 1, 2, .., 7]
        batches[i] = batch

    return batches

3. Deep Direct Reinforcement Network (DDRN)¶

3.1 Direct Reinforcement Network (DRN)¶

Deep Neural Network is able to provide a brilliant classifier or predict further possible outcome. Reinforcement Network otherwise can include the state so far, the action it may produce and the state plus the action will result to the next state as well as the reward for this action. The essay hereby combines the two into Deep Direct Reinforcement Network (DDRN), where Deep Neural Network works for prediction by the stock price and Reinforcement Network makes the action and stores the states and rewards for further iteration. The graph is showed below:

Action here is to adjust the investment distribution according to the output of the Neural Network, i.e. if the output is x, y, z and then the agent will distribute all assets (including capital and profits) with regard to x:y:z.

Reward is the profits we gain, and state is the stock log return during the last few days.

It is also noted that this DRL cell is like the squential situation, which also takes the input step by step and convey and iterate the state sequentially. Therefore, the essay implemented the DRL by a squential structure, with all states, actions and rewards updated with trading days going by. The codes are showed below:

def reinforcement(data, state, lag, cost, step, flag, testing=False):
    flag = flag.lower()
    if testing:
        num_stocks = data.shape[2]
        num_batches = data.shape[0]
    else:
        num_stocks = data.shape[2].value
        num_batches = data.shape[0].value
        
    actions = [np.reshape(np.ones(num_stocks) / num_stocks, [1, num_stocks])]# for calculation only, remove at last
    states = [state] # initial investment

    for i in range(step - 1):
        if flag == "cnn":
            output = convolution(data[:, i:lag+i+1, :], testing=testing)
        else:
            output = recurrent(data[:, i:lag+i+1, :], flag=flag, testing=testing)

        actions.append(dense(output, actions[-1], testing=testing)) # lag days and today (lagn, ..., lag2, lag1, 0)
        rewards = states[-1] * actions[-1] * tf.exp(data[:, i+lag+1, :]) - cost * tf.abs(actions[-1] - actions[-2])
        states.append(tf.reshape(tf.reduce_sum(rewards, axis=1), [num_batches, 1]))

    return tf.convert_to_tensor(actions[1:], tf.float64), states

def convolution(data, testing=False):
    if testing:
        num_stocks = data.shape[2]
        num_dates = data.shape[1]
    else:
        num_stocks = data.shape[2].value
        num_dates = data.shape[1].value

    with tf.variable_scope('cnn', reuse=tf.AUTO_REUSE):
        inputs = tf.reshape(data, [-1, num_dates, num_stocks, 1])

        # convolutional layer 1
        conv1 = tf.layers.conv2d(inputs=inputs, filters=12, kernel_size=[5, 5], strides=[1, 1], padding="same", activation=tf.nn.relu)
        pool1 = tf.layers.max_pooling2d(inputs=conv1, pool_size=[2, 2], strides=2, padding='SAME') # padding to make use of edges
        dropout1 = tf.layers.dropout(inputs=pool1, rate=0.3, training=True)

        # convlutional layer 2
        conv2 = tf.layers.conv2d(inputs=dropout1, filters=24, kernel_size=[5, 5], strides=[1, 1], padding="same", activation=tf.nn.relu)
        pool2 = tf.layers.max_pooling2d(inputs=conv2, pool_size=[2, 2], strides=2, padding='SAME')
        dropout2 = tf.layers.dropout(inputs=pool2, rate=0.3, training=True)
                                            # 10 -> 5 -> 3
        p2_flat = tf.reshape(dropout2, [-1, math.ceil(math.ceil(num_dates / 2) / 2) * \
                                            math.ceil(math.ceil(num_stocks / 2) / 2) * \
                                            24]) 

        # dense layer
        h1 = tf.layers.dense(inputs=p2_flat, units=1000, activation=tf.nn.relu)
        dropout3 = tf.layers.dropout(inputs=h1, rate=0.4, training=True)
        
        # output layer
        outputs = tf.layers.dense(inputs=dropout3, units=num_stocks, activation=None)

    return outputs

def recurrent(data, flag, testing=False):
    flag = flag.lower()
    
    if testing:
        num_dates = data.shape[1]
        num_stocks = data.shape[2]
    else:
        num_dates = data.shape[1].value
        num_stocks = data.shape[2].value

    with tf.variable_scope('rnn', reuse=tf.AUTO_REUSE): # choose different flag
        if flag == "lstm":
            cell = tf.contrib.rnn.BasicLSTMCell(num_stocks * 4)
            rnn, _ = tf.contrib.rnn.static_rnn(cell, tf.unstack(data, num_dates, axis=1), dtype=tf.float64)
        elif flag == "gru":
            cell = tf.contrib.rnn.GRUCell(num_stocks * 4)
            rnn, _ = tf.contrib.rnn.static_rnn(cell, tf.unstack(data, num_dates, axis=1), dtype=tf.float64)
        else:
            print("No this cell. The function only includes: LSTM, GRU.")

        h = tf.layers.dense(inputs=rnn[-1], units=256, activation=None)
        dropout = tf.layers.dropout(inputs=h, rate=0.4, training=True)
        
        outputs = tf.layers.dense(inputs=dropout, units=num_stocks, activation=None)

    return outputs

3.4 Deep Fully Connected Network (DFCN)¶

The essay not only takes the price or return as the inputs for DNN (Lin et al., 2013). Hereby we introduce a special process the essay undertakes to remain the most of previous information (state). For the output of CNN or RNN, it will be sent to the next DFCN, where the previous investment distribution among each stock is also input. The two combined as the input will result to the output with the length equals to the number of stocks. And these values then go through "softmax" so as to sum them to one. def dense(data, action, testing=False):
    if testing:
        num_stocks = data.shape[1]
    else:
        num_stocks = data.shape[1].value

    with tf.variable_scope('dense', reuse=tf.AUTO_REUSE):
        w1 = tf.get_variable("weightsforcnn", shape=[num_stocks, num_stocks * 2], dtype=tf.float64, initializer=tf.random_normal_initializer())
        b1 = tf.get_variable("biasforcnn", shape=[num_stocks * 2], dtype=tf.float64, initializer=tf.constant_initializer(0.0))
        wp = tf.get_variable("weightsforpolicy", shape=[num_stocks, num_stocks * 2], dtype=tf.float64, initializer=tf.random_normal_initializer())
        output1 = tf.sigmoid(tf.matmul(data, w1) + tf.matmul(action, wp) + b1) # combine the previous portfolio
        
        w2 = tf.get_variable("weights2", shape=[num_stocks * 2, num_stocks], dtype=tf.float64, initializer=tf.random_normal_initializer())
        b2 = tf.get_variable("bias2", shape=[num_stocks], dtype=tf.float64, initializer=tf.constant_initializer(0.0))
        output2 = tf.nn.softmax(tf.add(tf.matmul(output1, w2), b2))

        output2 = tf.nn.softmax(data)

        return output2

def runAgent(ax, label="", show_train=True):
    print(flag+label, "method works now...")
    tf.reset_default_graph()

    data = tf.placeholder(tf.float64, shape = [num_batches_train, lag + step, num_stocks])

    actions, states = reinforcement(data, tf.constant(np.full([num_batches_train, 1], 10000.0)), lag, cost, step, flag)
    optimizer = tf.train.AdamOptimizer(learning_rate=learning_rate).minimize(-tf.reduce_mean(states[-1]))

    test_batches = build_batch(test_data, step, lag, num_batches_test)
    actions_pred, states_pred = reinforcement(test_batches, tf.constant(np.full([num_batches_test, 1], 10000.0)),
                                   lag, cost, step, flag, testing=True)

    trains = [initial_invest]
    preds = [initial_invest]
    with tf.Session() as sess:
        sess.run(tf.global_variables_initializer())
        sess.run(tf.local_variables_initializer())
        for epoch in range(1, epochs+1):
            train_batches = build_batch(train_data, step, lag, num_batches_train)
            _, states_ = sess.run([optimizer, states], feed_dict = {data: train_batches})

            train = sum(states_[-1]) / num_batches_train # train
            pred = sum(states_pred[-1].eval()) / num_batches_test # test

            trains.append(train)
            preds.append(pred)

            if epoch%50 == 0 or epoch == 1:
                print("No.%d Epoch: Train: %0.2f ; Test: %0.2f"%(epoch, train, pred))
        actionss = actions_pred.eval()
     
    if show_train:
        ax.plot(trains, label='Train'+label)
    ax.plot(preds, label='Test'+label)
    ax.set_title(flag, fontsize=12)
    ax.legend()
    return actionss;

4. Deep Q Network (DQN)¶

DQN is famous for games where states, actions and rewards are all significant and clear and it is easier to learn Q values in Q learning. For the purpose of research, the essay attempts to utilize this model to build a trading agent, so as to test whether this model in stock market is as brilliant as in the video games.

The essay hereby introduces DQN as an agent-maker algorithm. Here it takes number of batches, number of dates (as lag days), number of stocks and other hyper parameters. And it will stores a deep learning model (the essay uses CNN here) to calculate the Q value of each action, for corresponding states. On the other words, DQN inputs the state (the prices of the stocks today and before) to DNN to get the Q values of every action. In algorithm, the off-policy rule makes value of the action decides the next action. But after some time, the state-reward-action sets during this time will be re-predict by the DNN, and the new values will be added to the old action value. This new-old value pair will be used to train the DNN again. This is the Experience Replay.

In conclusion, the difference between DQN and previous DDRN is the learning target. DQN learns the Q values, while DDRN learns to predict the stocks directly.

4.1 Agent Deployment¶

The essay takes the algorithm from Human-level control through deep reinforcement learning (Mnih et al., 2015) as below. For the sake of learning, I write it by keras API here to just save the model time. The experience replay algorithm is also showed. The memoryD stores the state-actoin-reward pairs to update the DNN.

class DQNAgent:
    def __init__(self, num_batches, num_dates, num_stocks, learning_rate, model_name=None, testing=False):
        self.num_batches = num_batches
        self.state_size = num_dates
        self.action_size = num_stocks
        self.testing = testing
        self.learning_rate = learning_rate
        
        self.gamma = 0.9
        self.epsilon = 0.95
        
        self.memoryD = []

        self.model = load_model(model_name) if testing else self._model()
        
    def _model(self):
        model = Sequential()
        model.add(keras.layers.Conv2D(12, (5, 5), padding="same",
                                      input_shape=(self.state_size, self.action_size, 1), activation="relu"))

        model.add(keras.layers.MaxPooling2D(pool_size=(2, 2), strides=2, padding='SAME'))
        model.add(keras.layers.Dropout(0.25))

        model.add(keras.layers.Conv2D(24, (5, 5), padding="same", 
                                      input_shape=(math.ceil(self.state_size / 2), math.ceil(self.action_size / 2), 1), activation="relu"))

        model.add(keras.layers.MaxPooling2D(pool_size=(2, 2), strides=2, padding='SAME'))
        model.add(keras.layers.Dropout(0.25))

        model.add(keras.layers.Flatten())
        model.add(keras.layers.Dense(1024, activation="relu"))
        model.add(keras.layers.Dropout(0.2))
        
        model.add(Dense(self.action_size * 2 + 1, activation="linear"))
        model.compile(loss="mse", optimizer=Adam(lr=self.learning_rate))

        return model

    def act(self, state):
        if not self.testing and random.random() - self.epsilon > 0:
            return np.argmax([np.random.dirichlet(np.ones(self.action_size)) for i in range(self.num_batches)], axis=1)
        else:
            return np.argmax(self.model.predict(state), axis=1)

    def expReplay(self):
        for state, action, reward, state_p, done in self.memoryD:
            y = reward + self.gamma * np.max(self.model.predict(state_p)) * (not done)
            y_all = self.model.predict(state)
            y_all[:, action] = y
            self.model.fit(state, y_all, verbose=0)
        self.memoryD.pop(0)

4.2 Trading Policy¶

States and actions are easy to represent since price and return are determined. But actions can vary a lot. Since Q value table cannot be too continuous in terms of actions though DNN can ease a little. I hereby give $(numberofstocks * 2 + 1)$ as the action size. The first $numberofstocks$ actions represent investing this stocks, the second $numberofstocks$ actions represent withdrawing investment, and the last one action represents doing nothing.

def runDQN():
    tf.reset_default_graph()
    agent = DQNAgent(num_batches_train, lag+1, num_stocks, learning_rate)
    final_train = []
    for epoch in range(1, epochs+1):
        batches = build_batch(train_data, step, lag, num_batches_train)
        invest_all = initial_invest * np.ones(num_batches_train)
        pre_invest = np.array([[initial_invest / num_stocks for i in range(num_stocks)] for _ in range(num_batches_train)])
        for i in range(step-1):
            state = batches[:, i:i+lag+1, :].reshape(num_batches_train, lag+1, num_stocks, 1)
            action = agent.act(state)
            state_p = batches[:, i+1:i+lag+2, :].reshape(num_batches_train, lag+1, num_stocks, 1)

            for batch in range(num_batches_train): # policy for each batch
                one_invest = invest_all[batch] * 0.2
                if action[batch] < num_stocks:
                    pre_invest[batch] = [invest - one_invest / num_stocks * (action[batch]!=stock) + one_invest * (action[batch]==stock) \
                                         for stock, invest in enumerate(pre_invest[batch])]
                elif num_stocks <= action[batch] < num_stocks * 2:
                    pre_invest[batch] = [invest + one_invest / num_stocks * (action[batch]!=stock) - one_invest * (action[batch]==stock) \
                                         for stock, invest in enumerate(pre_invest[batch])]

            reward = np.sum((np.exp(state_p[:, -1, :, :].squeeze()) - 1) * pre_invest, axis=1) - cost * invest_all * 0.2
            invest_all = invest_all + reward

            done = (i == (step-2))
            agent.memoryD.append((state, action, reward, state_p, done))                   
            state = state_p

            if len(agent.memoryD) >= 10:
                agent.expReplay()

        agent.memoryD = []
        final_train.append(np.mean(invest_all))

        print("No.%d Epoch: Train: %0.2f"%(epoch, final_train[-1]))
        if epoch % 10 == 0:
            model_name = "model" + str(epoch)
            agent.model.save(model_name)

4.3 Discuss¶

The return will imporve after long time training, but too slowly, which has been testified, but cannot be showed here due to the composition. The results for DQN agent are not very satisfactory, this is actually a drawback of DQN that it cannot monitor the time series very well, since time series data has many possible rules, i.e. the Q-values of the action is hard to monitor automatically with this Experience Replay Zheng et al., 2016; Zheng et al., 2014; Lngkvis et al., 2014):

  • Firstly, the model needs the overall understanding towards the actions. For example, video games do limited actions that can be calculated for each Q values. However, it is not easy as well to define the actions in stock market. Buying or selling has differing forms, with the amount, the time, the exchange and so on to concern. So if the action policy is not stable or practical, the agent will unlikely lead to global optimization. E.g. if we use DQN to play Mario, only telling the agent that player can jump, without saying one can also jump high and jump low. The agent will learn, but it is too slow.

  • Secondly, unlike games can be played again and again to train by themselves when restarting the games. The stock market data is actually limited and the amount is not big at all to excute the long-time training. Thus, the environment for agent to learn is not quite enough.

  • Lastly, a game always have a rule since it is made by people, but stock market rule is even harder to confirm whether there is a rule and whether this rule is useful next time.

In spite of them, with my policy hereby, the agent can learn to perform better to achieve more profits, though very slowly.

5. Conclusion¶

The essay makes two different kinds of agents, i.e. DDRN Agent and DQN Agent. I hereby compare the two for further understanding of them.

DDRN uses a direct reinforcement network as the foundation, while DQN uses Q learning reinforcement. The former makes sequential deep neural network, with each state holds a deep neural network to be responsible for prediction. The latter makes the sequential network as well, but uses the neural network to acquire Q value for each state and action.

For DDRN, my action is the distribution of the stocks by investment. For DQN, my action is to buy or to sell the stocks. The former is more flexible and tend to find the optimization since its large coverage. The latter highly relies on the action we can choose.

For DDRN, the essay provides a good agent to help make over 14% within 100 trading days, and even approaching the best policy. At the same time, the model itself is proven to be robust, overall and practically to many situations. As for DQN, the essay provides a solution creatively and opens a good mind towards trading with constraints.

The work provides good results in allocation trading or portfolio formation. Further focus on action-trading and how to provide specific actions towards different issues is in need for further research.
