CIFAR10 Image Classification¶

Tianzhu Qin¶

The essay compares Gradient Descent, AdaGrad, Nesterov, Adam by CNN towards CIFAR10 with both Dropout and Non-Dropout tried respectively.

0. Dataset¶

SOURCE: https://www.cs.toronto.edu/~kriz/cifar.html

The CIFAR-10 dataset consists of 60000 32x32 colour images in 10 classes, with 6000 images per class. There are 50000 training images and 10000 test images.

The dataset is divided into five training batches and one test batch, each with 10000 images. The test batch contains exactly 1000 randomly-selected images from each class. The training batches contain the remaining images in random order, but some training batches may contain more images from one class than another. Between them, the training batches contain exactly 5000 images from each class.

1. Implementation¶

A CNN architecture that consists of 3 convolutional layers followed by a hidden fully connected layer of 1000 units.

Each convolutional layer consists of a sublayer of 5x5 convolutional filters with stride 1 followed by a sublayer of 2x2 max-pool units with stride 2. Each neuron applies ReLU activation function.

The hyper-parameter settings hereby:

minibatch size = 128

learning rate = 0.001

total number of epoches = 100

import numpy as np
import pandas as pd
import tensorflow as tf
import matplotlib.pyplot as plt
from keras.datasets import cifar10

(data_train, label_train), (data_test, label_test) = cifar10.load_data()

data_train = data_train / 255 # get float32 for conv2d and metrics
data_test = data_test / 255
label_train =np.squeeze(label_train) # squeeze the 1
label_test =np.squeeze(label_test)

tf.reset_default_graph()
tf.logging.set_verbosity(tf.logging.INFO)

def classification_cnn(data, training=False):
    with tf.variable_scope('classification_cnn', reuse=tf.AUTO_REUSE):
        # 32*32*3 for rgb
        inputs = tf.reshape(data, [-1, 32, 32, 3])
        # convolutional layer 1
        conv1 = tf.layers.conv2d(inputs=inputs, filters=32, kernel_size=[5, 5], strides=[1, 1], padding="same", activation=tf.nn.relu)
        # pooling layer
        pool1 = tf.layers.max_pooling2d(inputs=conv1, pool_size=[2, 2], strides=2) # stride 2
        dropout1 = tf.layers.dropout(inputs=pool1, rate=0.3, training=training)

        # convlutional layer 2
        conv2 = tf.layers.conv2d(inputs=dropout1, filters=64, kernel_size=[5, 5], strides=[1, 1], padding="same", activation=tf.nn.relu)
        pool2 = tf.layers.max_pooling2d(inputs=conv2, pool_size=[2, 2], strides=2)
        dropout2 = tf.layers.dropout(inputs=pool2, rate=0.3, training=training)

        # convlutional layer 2
        conv3 = tf.layers.conv2d(inputs=dropout2, filters=128, kernel_size=[5, 5], strides=[1, 1], padding="same", activation=tf.nn.relu)
        pool3 = tf.layers.max_pooling2d(inputs=conv3, pool_size=[2, 2], strides=2)
        dropout2 = tf.layers.dropout(inputs=pool3, rate=0.3, training=training)
        
        p2_flat = tf.reshape(dropout2, [-1, 4 * 4 * 128]) # 32 / 2 / 2 / 2

        # dense layer
        h1 = tf.layers.dense(inputs=p2_flat, units=1000, activation=tf.nn.relu)
        dropout3 = tf.layers.dropout(inputs=h1, rate=0.4, training=training)
        
        # output layer
        outputs = tf.layers.dense(inputs=dropout3, units=10, activation=None)
       
    return outputs

def build_batches(batch_size, data, labels, num_batch):
    """ input batch size, the whole data, the whole labels, and the number of batches that need
        return the batches
    """
    id_shuffle = np.arange(0 , len(data))
    np.random.shuffle(id_shuffle) # shuffle the id rather than data in order to save memory
    batches = []

    for i in range(num_batch):
        id_sample = id_shuffle[batch_size*i:batch_size*(i+1)] # get the shuffled data
        data_batch = [data[i] for i in id_sample]
        labels_batch = [labels[i] for i in id_sample]
        batches.append((np.array(data_batch), np.squeeze(np.array(labels_batch))))

    return batches
      
def run_cnn(optimizer_flag, training=False):
    # set params
    print('\nStart', optimizer_flag)
    batch_size = 128
    x_dim = 32
    x_channel = 3
    learning_rate = 0.001
    num_epoches = 100
    num_batches = 50

    # set scope
    with tf.variable_scope('main', reuse=tf.AUTO_REUSE):
        x = tf.placeholder(tf.float64, [batch_size, x_dim, x_dim, x_channel]) # 128 * 32 * 32 * 3
        y = tf.placeholder(tf.int64, [batch_size, ]) # 128,
        outputs = classification_cnn(x, training=training)
        loss = tf.reduce_mean(tf.nn.sparse_softmax_cross_entropy_with_logits(labels=y, logits=outputs)) * 100.0
        if optimizer_flag == "GD":
            optimiser = tf.train.GradientDescentOptimizer(learning_rate).minimize(loss)
        elif optimizer_flag == "AdaGrad":
            optimiser = tf.train.AdagradOptimizer(learning_rate).minimize(loss)
        elif optimizer_flag == "Nesterov":
            momentum = 0.7
            optimiser = tf.train.MomentumOptimizer(learning_rate, momentum).minimize(loss)
        elif optimizer_flag == "Adam":
            optimiser = tf.train.AdamOptimizer(learning_rate).minimize(loss)

        preds = classification_cnn(data_test, training=False) # predict the result
        predictions_idx = tf.argmax(tf.nn.softmax(preds), axis=1)
    
    with tf.Session() as sess:
        sess.run(tf.global_variables_initializer())
        sess.run(tf.local_variables_initializer()) # initial also local
        loss_list = []
        for epoch in range(1, num_epoches+1):
            batches = build_batches(batch_size, data_train, label_train, num_batches) # build batches
            loss_list_temp = []
            for batch_data, batch_labels in batches:
                _, l = sess.run([optimiser, loss], feed_dict={x: batch_data, y: batch_labels})
                loss_list_temp.append(l)
            loss_mean = np.mean(loss_list_temp) # average the loss
            loss_list.append(loss_mean)
            loss_list_temp = []
            if epoch % 20 == 0: 
                print('Epoch: {}, Loss: {:.4f}'.format(epoch, loss_mean))
            
        accuracy = sum(predictions_idx.eval() == label_test) / len(label_test) # calculate the accuracy
        print("%s accuracy: %f"%(optimizer_flag, accuracy))
            
    return loss_list

def plot_data(lst1, lst2, lst3, lst4, ax, title):
    """input the different list for each optimiser, axe that use, title name of figure
    """
    df_results = pd.DataFrame({"GD": lst1, "AdaGrad": lst2, "Nesterov": lst3, "Adam": lst4})
    df_results.index = np.arange(1, len(lst1)+1)
    df_results.plot(fontsize=15, ax=ax)
    ax.set_title(title, fontsize=17)
    ax.set_xlabel("Number of Epochs", fontsize=15)
    ax.set_ylabel("Loss", fontsize=15)
    ax.legend(bbox_to_anchor=(1.001, 1));
    
if __name__ == "__main__":
    fig, axes = plt.subplots(ncols=2, nrows=1, figsize=(17, 7))

2. Optimiser Comparation¶

Gradient Descent is the normal optimiser updating parameters reponding to the direction of the gradients with the learning rate determining how big. But it does not always converge to a solid result, and the learning rate also decide its speed and may not converge within some epochs if it is too slow, as showed in the graph.

AdaGrad modifies learning rate by its feature frequency. The more frequent the larger the updates, and vice versa. So unbalanced sparse data will be fixed. However our image data does not really have sparse issue, so here in the graph it is not significant.

Nesterov uses momentum term to give constant influence from the direction before. And in this case, converge can be sped up and the direction is also including the previous information, which is more likely to be correct. So we can find it is better than just gradient descent.

Adam utilizes not only decay average of past gradients like Nesterov, but also the past squared gradients, so both first moment (mean) and second moment (variance) of the information are stored which brings the best results among the four.

3. Result Discussion¶

The graph presents the average loss (which multiplies 100 to make graph clear and converge well) per epoch for the four different optimisers before and after dropout.

The left and the right do share similar pattern. Although without dropout, the model gives a lowere loss value. For Adam, Gradient Descent and Nesterov, the three do hold similar results towards the end. But with dropout, Gradient Descent and Nesterov just cannot converge to the same point with Adam. It is the same to see in accuracy that without dropout, the accuracy of the three is nearer than that with dropout. In conclusion, although dropout makes the line smoother, which reduce overfitting, it turns out to give a more bias result away from the model will get without dropout.

From the graph with dropout, we can see that Adam is the best for after 100 epochs, but this time the four do not have too many differences from the first question, which may due to the larger dataset and longer iteration. The four optimisers all converge fast at very first, but AdaGrad gets slower quite earily. And it is the worst one after all. Besides, trends of Nesterov and gradient are quite similar and their values for loss even converge to near points. My accuracy calculation from Top 1 Score also shows similar results. Adam has the highest accuracy above 0.75. Gradient descent is just about 0.68, which is very near to Adam. Nesterov is about 0.66; however AdaGrad only has a value around 0.54.

Although all accuracy this time rise up than the last question, the relative performances of these four optimisers are quite similar. But here Gradient descent and Nesterov is even closer than before. On the other hand, dropout in the model smooth the result but make it even bias.
